redis持久化方法

目前Redis持久化的方式有两种： RDB 和 AOF

RDB持久化

通过配置持久化策略实现

# 在几秒内改动了多少数据就触发持久化
# 想禁用的话不设置save即可
save 900 1   # 参数的设置具体为: 如果每900秒有1个键进行了改动则保存一次数据
save 300 10
save 60 10000

# 备份进程出错 主进程停止写入操作
stop-writes-on-bgsave-error yes

# 是否压缩rdb文件 
# 其实就相当于说是要牺牲CPU还是牺牲硬盘空间,大多数情况下都是后者,因此默认配置NO就可以了 
rdbcompression no

也可以通过手动执行命令 save/bgsave 实现持久化.

每次执行会重新生成一个快照文件(dump.rdb).

save是同步执行. bgsave是异步执行

AOF (append only file)

通过日志文件进行数据的恢复

其实这个方法的话,和mysql的二进制文件(binlog)去恢复丢失数据有点类似

都是将我们对redis的每一项操作写入到日志文件中

# 修改配置文件来打开aof功能：
appendonly yes

# 该参数的配置有三种选择
# always   每次有新命令追加到aof文件时就执行一个持久化，比较慢但是安全
# everysec 每秒执行一次持久化,相当于在故障的时候只会丢失1s的数据
# no       不进行持久化
appendfsync [param:always/everysec/no]
